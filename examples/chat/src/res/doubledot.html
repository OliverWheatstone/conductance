Double-dots

SJS adds the .. ("double-dot") syntax as a convenient way to use functional APIs while maintaining the familiar look and chainaing ability of Object-Oriented APIs.

Essentially, `a .. fn(b, c)` transforms into `fun(a, b, c)`. Since a lot of functions in conductance (and SJS) take their "subject" as a first argument, this allows us to extend objects unobtrusively. e.g to iterate over an array, you can do: 

[1, 2, 3] .. map(elem -> elem \* 2)

Which actually results in the following call:

map([1, 2, 3], elem -> elem \* 2)

We could have added the `map` method to Array.prototype, but using a simple function allows `map` to work on multiple types of objects (arrays, infinite streams and `arguments` objects). Using plain functions also means we don't have to fight with other libraries that might also want to add their own Array functions.
